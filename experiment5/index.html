Description

In this programming assignment, we're building a generator that directly reads inspiring images, renders parameterized images, and adjusts its own designs to optimize an evaluation score. In our implementation, we have not only included multiple different shape types to demonstrate different executions of this iteration, but we have also included one implementation that works in full color!

While the program will eventually stumble into a satisfying solution on its own, it is recommended that the user be mindful of the selected mutation rate. At first, a high rate is ideal for finding specific permutations that better replicate the original. Yet, as you get closer to a viable output, it is best to decrease this permutation rate in order to focus your search just to a local minimum.

Technical

Our implementation starts out by populating a number of shape objects into an array, all of which are provided with a random starting position and color. Then, through a function that is able to register how "similar" this new image is to the desired one, the program is then able to store whichever permutation received the closest resemblance score, and iterate from there

Just how different following iterations will be from the current closest image is determined by a user-selected mutation rate, which allows for the program to differ as much or as little from the current minimum value accordingly.

One noteworthy deviation from this implementation can be observed in the color image, in which the program can no longer simply mutate within the range of 0 - 255 (Black - White). In this instance,the program instead assigns each line within the image a random color value that is picked from the inspiring image. This allows for a color range that still reacts to mutation values, as the amount of variation in these referential coordinates changes based on the same input.
